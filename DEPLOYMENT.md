# RAAS Deployment Guide

Deploy frontend (Vercel), backend (Render or Azure), database (Neon or Supabase), and email (Resend).

---

## Architecture Overview

```
┌─────────────┐     HTTPS      ┌──────────────┐     SQL      ┌─────────────┐
│   Vercel    │ ──────────────▶│ Render/Azure │ ────────────▶│ Neon/       │
│  Angular    │   /api calls   │  .NET 8 API  │              │ Supabase    │
└─────────────┘                └──────┬───────┘              └─────────────┘
                                      │
                                      ▼ HTTPS
                               ┌──────────────┐
                               │   Resend     │
                               │   (emails)   │
                               └──────────────┘
```

## Step 1 — Database (Neon or Supabase)

### Option A: Neon (recommended, free tier)

1. Go to [neon.tech](https://neon.tech) → Create project `raas-db`
2. Copy the **connection string** (Pooled connection recommended)
3. Append if missing: `;SSL Mode=Require;Trust Server Certificate=true`
4. Example:
   ```
   Host=ep-cool-name-123456.us-east-2.aws.neon.tech;Port=5432;Database=raas_db;Username=raas_owner;Password=SECRET;SSL Mode=Require;Trust Server Certificate=true
   ```
5. **Optional:** Run `database/schema.sql` in Neon SQL Editor (EF migrations also run on API startup)

### Option B: Supabase

1. Go to [supabase.com](https://supabase.com) → New project
2. **Settings → Database** → copy connection string (URI mode)
3. Convert to Npgsql format:
   ```
   Host=db.xxxxx.supabase.co;Port=5432;Database=postgres;Username=postgres;Password=YOUR_PASSWORD;SSL Mode=Require;Trust Server Certificate=true
   ```
4. **Storage (product images):**
   - Storage → New bucket `product-images` → set **Public**
   - Upload images → copy public URL → paste in Admin → Product → Image URL

## Step 2 — Email (Resend)

1. Sign up at [resend.com](https://resend.com)
2. **API Keys** → Create key → copy `re_xxxxxxxx`
3. **Domains** → Add your domain (e.g. `raas.in`) → add DNS records
   - Until verified, use Resend sandbox: `onboarding@resend.dev` (only sends to your account email)
4. Set backend env vars:

| Variable | Example |
|----------|---------|
| `Resend__ApiKey` | `re_abc123...` |
| `Resend__FromEmail` | `RAAS <orders@raas.in>` |
| `Resend__AdminEmail` | `admin@raas.in` |
| `Resend__Enabled` | `true` |

### Email triggers (automatic)

| Event | Trigger location | Recipient |
|-------|------------------|-----------|
| User registered | `AuthService.RegisterAsync` | Customer |
| Order placed | `OrderService.PlaceOrderAsync` | Customer + Admin |
| Order shipped | Admin updates status → Shipped | Customer |
| Order delivered | Admin updates status → Delivered | Customer (+ referral code) |
| Low stock (≤10) | `OrderService.PlaceOrderAsync` | Admin |

Emails use branded HTML templates in `src/RAAS.Infrastructure/Email/EmailTemplates.cs`.

**Local dev:** Leave `Resend__ApiKey` empty — emails are logged to console, not sent.

## Step 3 — Backend (Render — free tier)

1. Push code to **GitHub**
2. Go to [render.com](https://render.com) → **New → Blueprint**
3. Connect repo → Render reads `render.yaml`
4. Set these **secret** env vars in Render dashboard:

| Key | Value |
|-----|-------|
| `ConnectionStrings__DefaultConnection` | Your Neon/Supabase connection string |
| `Resend__ApiKey` | `re_xxx` |
| `Resend__FromEmail` | `RAAS <orders@raas.in>` |
| `Resend__AdminEmail` | your email |
| `Frontend__AllowedOrigins__0` | `https://YOUR-APP.vercel.app` |
| `Jwt__Key` | Auto-generated by Render (or set 32+ char secret) |

5. Deploy → note your API URL: `https://raas-api.onrender.com`
6. Verify: `https://raas-api.onrender.com/health`

### Option B: Azure App Service

```powershell
# Publish locally
dotnet publish src/RAAS.Api -c Release -o ./publish

# Create App Service (Portal or CLI), then set Application Settings:
# ConnectionStrings__DefaultConnection
# Jwt__Key, Resend__ApiKey, Resend__FromEmail, Resend__AdminEmail
# Frontend__AllowedOrigins__0 = https://your-app.vercel.app
```

Upload `publish/` folder via Zip Deploy or GitHub Actions.

### Docker (any host)

```bash
docker build -t raas-api .
docker run -p 8080:8080 \
  -e ConnectionStrings__DefaultConnection="Host=..." \
  -e Resend__ApiKey="re_xxx" \
  -e Frontend__AllowedOrigins__0="https://your-app.vercel.app" \
  raas-api
```

## Step 4 — Frontend (Vercel)

1. Push code to **GitHub**
2. [vercel.com](https://vercel.com) → **Add New Project** → import repo
3. Configure:
   - **Root Directory:** `frontend`
   - **Build Command:** `npm run build` (runs `set-env.js` then `ng build`)
   - **Output Directory:** `dist/frontend/browser`
4. **Environment Variables:**

| Name | Value |
|------|-------|
| `API_URL` | `https://raas-api.onrender.com/api` |

5. Deploy → note URL: `https://your-app.vercel.app`
6. **Update backend CORS:** Add your Vercel URL to `Frontend__AllowedOrigins__0` on Render/Azure → redeploy API

---

## Step 5 — Connect everything (checklist)

- [ ] Database running, API `/health` returns 200
- [ ] API `GET /api/products/categories` returns data
- [ ] Vercel `API_URL` points to live API (with `/api` suffix)
- [ ] CORS: Vercel URL in `Frontend__AllowedOrigins__0`
- [ ] Resend API key set, domain verified (or sandbox for testing)
- [ ] Login works: `admin@raas.in` / `Admin@123`
- [ ] Place test order → check Resend dashboard for sent emails
- [ ] Admin order status → Shipped/Delivered triggers emails

## Environment variable reference (.NET)

ASP.NET Core maps env vars with `__` for nested config:

```
ConnectionStrings__DefaultConnection
Jwt__Key
Jwt__Issuer
Jwt__Audience
Resend__ApiKey
Resend__FromEmail
Resend__AdminEmail
Resend__Enabled
Frontend__Url
Frontend__AllowedOrigins__0
Frontend__AllowedOrigins__1
EnableSwagger
ASPNETCORE_ENVIRONMENT=Production
```

Copy `.env.example` to `.env` for local reference (never commit secrets).

## Troubleshooting

| Problem | Fix |
|---------|-----|
| CORS error in browser | Add exact Vercel URL to `Frontend__AllowedOrigins__0`, redeploy API |
| API 502 on Render | Check logs; usually bad DB connection string |
| DB SSL error | Add `SSL Mode=Require;Trust Server Certificate=true` |
| Emails not sending | Verify `Resend__ApiKey`; check Resend dashboard → Logs |
| Resend sandbox | Only sends to verified email until domain is added |
| Frontend calls wrong API | Set `API_URL` in Vercel, redeploy frontend |
| Migrations fail | Ensure DB exists; check connection string host/password |
